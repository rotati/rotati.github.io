Duration: 2 days

Course Description:

Introduction to Domain-Driven Design
Improving Applications Design with a Rich Domain Model

Most modern systems and applications are developed using object-oriented languages. Object-orientation (OO) is chosen to better tackle complexity. But most complex enterprise Java applications are written using a procedural style. This course aids participants in developing a richer domain model using Domain-Driven Design (DDD) techniques, and balance between the business domain and addressing technological issues.

This training course uses lecture, discussion, and classroom exercises, including hands-on coding projects. In discussions, participants have the opportunity to discuss their own experiences, and relate their own work to the principles and techniques presented in the class. All code exercises are written in Java.

This class was developed and is taught by some of the World' well-known and respected Java software engineers and architects, and uses the format trusted by some of the best companies in the World.

Audience:

Software designers, business analysts, project managers, programmers and developers as well as operational managers and software division managers.

Prerequisite:

Basic knowledge of object modeling and design. Knowledge of UML and any object-oriented language like C++, C#, Java, VB.NET would be a plus.
